Superposition

Track 3: Interpretability
5 of 8
MLP Mechanics, Circuits
80 min
Text · Vision · Speech

Learning Outcomes

  • Explain the superposition hypothesis and why it arises in neural networks
  • Distinguish polysemanticity (neuron level) from superposition (geometry level)
  • Describe how Sparse Autoencoders (SAEs) recover monosemantic features
  • Explain the geometry of superposition: how features are packed as nearly-orthogonal vectors
  • Interpret the sparsity-capacity tradeoff and predict when superposition is likely
  • Evaluate SAE reconstruction quality using L0, L1, and explained variance metrics

Conceptual Setup

The superposition hypothesis, introduced by Elhage et al. 2022, proposes that neural networks represent more features than they have dimensions by encoding features as nearly orthogonal directions in activation space rather than as individual neurons. A network with d neurons can represent up to O(d / sparsity) features when those features are sparse — i.e., only a small fraction are active at any given time.

This explains a long-standing puzzle in interpretability: why are individual neurons polysemantic? A neuron in GPT-2 layer 9 might activate for "cat", "dog", "France", and "logarithm" — apparently unrelated concepts. Under superposition, this is not a bug but a consequence of efficient packing: each "true" feature is distributed across multiple neurons, and each neuron participates in multiple features.

Superposition creates a fundamental challenge for interpretability: the neuron basis is not the feature basis. We cannot simply look at what each neuron does. We must find the features — the actual computational units — which live as directions in activation space. Sparse Autoencoders (SAEs) are designed to do exactly this.

Core tension

Models need to represent more concepts than they have dimensions. Superposition is the solution — but it makes interpretation harder because no single neuron = no single concept.

Intuition First

Think of a city's street map. A city has thousands of buildings, but streets run in only a few directions (north-south, east-west, and diagonals). Each building is located using a combination of street coordinates. This is superposition: many buildings (features) encoded using few directions (dimensions). Two nearby buildings share coordinate components — but their full address distinguishes them.

The key requirement for superposition to work is sparsity: at any given time, only a few features are active. If every feature were always on, they would interfere catastrophically. With sparsity, the cross-feature interference averages to near zero.

Sparse Autoencoders are the tool that "de-superpose" activations. An SAE takes a d-dimensional activation vector, expands it to a much larger space (e.g., 4d or 64d), applies a sparsity-inducing activation (ReLU or TopK), then reconstructs the original. The expanded representation isolates individual features into individual dimensions of the wider space.

Technical Core

Superposition geometry: features are vectors f_i ∈ ℝ^d with ||f_i|| ≈ 1. If there are N features and N >> d, they must be nearly orthogonal but not perfectly orthogonal. The interference between features i and j is (f_i · f_j)². With N features in d dimensions, the mean squared interference ≈ N/d². For sparsity p (fraction of features active), the expected loss from interference ≈ p × N/d².

Toy model: d=2, N=5 features
Feature directions (nearly orthogonal in 2D):
  f1 →  [1.0,  0.0]
  f2 →  [0.6,  0.8]
  f3 →  [-0.6, 0.8]
  f4 →  [-1.0, 0.0]
  f5 →  [0.0, -1.0]

With sparsity=0.1, only ~1 feature active at a time:
  interference ≈ 0.1 × 5/4 = 0.125  (manageable)
With sparsity=0.5:
  interference ≈ 0.5 × 5/4 = 0.625  (severe)

Sparse Autoencoder architecture:

Input: x ∈ ℝ^d  (MLP activation at one token position)
   ↓
Encoder: h = ReLU(W_enc · (x - b_pre) + b_enc)  ∈ ℝ^N  (N >> d)
   ↓
Sparse h: most entries zero, few active
   ↓
Decoder: x̂ = W_dec · h + b_pre
   ↓
Loss = ||x - x̂||² + λ||h||₁
         reconstruction    sparsity penalty

SAE training objective: minimise reconstruction loss while encouraging sparsity (L1 penalty on hidden activations). The decoder columns W_dec[:,i] are the "feature directions" — each is a d-dimensional vector in activation space.

Dictionary Learning connection

SAEs are a form of dictionary learning: they find an overcomplete dictionary of features such that any activation can be represented as a sparse combination of dictionary atoms. This is related to ICA (Independent Component Analysis) and sparse coding (Olshausen & Field 1996).

Worked Example

import torch
import torch.nn as nn

# Minimal SAE implementation
class SparseAutoencoder(nn.Module):
    def __init__(self, d_model, d_hidden, l1_coeff=1e-3):
        super().__init__()
        self.W_enc = nn.Linear(d_model, d_hidden, bias=True)
        self.W_dec = nn.Linear(d_hidden, d_model, bias=True)
        self.l1_coeff = l1_coeff
        # Normalise decoder columns to unit norm
        with torch.no_grad():
            self.W_dec.weight.data = nn.functional.normalize(
                self.W_dec.weight.data, dim=0)

    def forward(self, x):
        h = torch.relu(self.W_enc(x))       # sparse hidden
        x_hat = self.W_dec(h)               # reconstruction
        recon_loss = (x - x_hat).pow(2).mean()
        l1_loss    = self.l1_coeff * h.abs().mean()
        return x_hat, h, recon_loss + l1_loss

# Usage: extract MLP layer 9 activations from GPT-2 on a corpus,
# then train the SAE on those activations.
# d_model=768, d_hidden=768*8=6144 is a common choice.
sae = SparseAutoencoder(d_model=768, d_hidden=6144)
print(f"SAE parameters: {sum(p.numel() for p in sae.parameters()):,}")
# 9,447,936 — about 8% of GPT-2-small

Mechanistic Interpretation Layer

Text Models

Anthropic's large-scale SAE analysis of Claude 3 Sonnet (Templeton et al. 2024) found SAE features that are strikingly monosemantic: single features active for "DNA sequences", "the concept of bridge in music", "racist tropes", "the Golden Gate Bridge". One feature activated strongly for thousands of contexts involving the Golden Gate Bridge across languages, modalities, and contexts. Steering the model with this feature caused the model to describe itself as the Golden Gate Bridge — demonstrating causal influence on generation.

Vision Models

Park et al. 2023 applied dictionary learning to ViT activations and found that SAE features in later layers correspond to interpretable visual concepts: "dog ear", "red stripe", "window frame." Early layers yield texture and frequency features. The degree of monosemanticity increases with layer depth, consistent with the text findings.

Speech Models

Superposition in speech models has been studied in wav2vec2 and HuBERT. Research shows that hidden units in these models are polysemantic at the neuron level but that sparse linear decompositions yield features corresponding to phonemic categories, prosodic features, and speaker identity. The sparsity is lower in speech models than text models — audio frames carry dense, continuous information unlike discrete tokens.

Hands-on Exercise

Visualise Superposition in a Toy Model

  • 1Implement the toy model from Elhage et al. 2022: a 1-layer network with ReLU, input dim=5, hidden dim=2. Train on random sparse binary vectors (sparsity=0.1).
  • 2After training, plot the 5 weight vectors W[:,i] in 2D. Observe that they form a pentagon — equally spaced near-orthogonal directions.
  • 3Repeat with sparsity=0.5. Observe that fewer features are represented — some collapse to zero.
  • 4Train an SAE (d=2, d_hidden=10) on the hidden activations of the trained toy model. Check if the SAE recovers the original 5 features.

Expected output: With sparsity=0.1, the toy model should represent all 5 features in 2D superposition. With sparsity=0.5, it should represent at most 2-3 features. The SAE should recover near-perfect features with >0.9 cosine similarity to the ground truth.

Reflection: Why does increasing sparsity allow more features to be stored? What is the geometric intuition?

Stretch: Train the SAE with L1 coefficient ranging from 1e-5 to 1e-1. Plot L0 vs explained variance as a Pareto curve. Identify the optimal tradeoff point.

Real Model Artifact

Templeton et al. 2024 "Scaling and evaluating sparse autoencoders" (Anthropic) trained SAEs on Claude 3 Sonnet residual stream with expansion factor 8× (d=4096 → N=32768 features). Key findings: (1) 73% of features had a clear human-interpretable label; (2) dead features: <2% after training with auxiliary loss; (3) mean L0 ≈ 25 (only 25 of 32768 features active per token); (4) explained variance >0.95; (5) features include multimodal concepts (same feature activates for the word "cat" and images of cats). The SAE weights and analysis tools are publicly available via Anthropic's interpretability team.

Assessment Check

Scenario 1

You train an SAE on GPT-2 layer 9 MLP activations and find that 40% of features are "dead" (never activate on your evaluation set). What are the most likely causes and how would you fix each?

Strong answer: (1) L1 coefficient too high → features get suppressed during training. Fix: reduce λ. (2) Decoder columns are not unit-normalised → scale degeneracy causes some features to dominate. Fix: add norm constraint. (3) Evaluation set is too small/homogeneous → features that activate on rare inputs appear dead. Fix: increase evaluation set diversity. (4) d_hidden too large for the amount of training data → underdetermined. Fix: reduce d_hidden or train longer.

Scenario 2

An SAE has high explained variance (0.97) but low feature interpretability (only 30% of features have clear labels). Is the SAE useful for interpretability?

Strong answer: High explained variance means the SAE captures the variance faithfully, which is necessary but not sufficient for interpretability. Low label rate could mean: (1) many features are real but encode concepts that humans don't have ready labels for (e.g., syntactic microfeatures); (2) many features are "junk" — artifacts of SAE training that don't correspond to model-used concepts; (3) the labelling method (max-activating examples) is too coarse. The SAE is partially useful: the 30% interpretable features can still be used for steering and circuit analysis. Test usefulness by checking if the interpretable features have causal effects consistent with their proposed labels.

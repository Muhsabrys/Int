Feature Visualization

Track 3: Interpretability
6 of 8
Superposition, Hidden States
70 min
Text · Vision · Speech

Learning Outcomes

  • Explain max-activating dataset examples and their limitations
  • Describe gradient-based feature visualisation for CNNs and ViT
  • Apply logit lens and feature steering to characterise a text model feature
  • Explain why activation maximisation can produce adversarial-looking inputs
  • Use automated interpretability (e.g., LLM-labelling of max-activating examples) correctly
  • Identify at least two failure modes of feature visualisation methods

Conceptual Setup

Feature visualisation answers the question: what does this feature detect? Once you have identified a direction in activation space (a neuron, an SAE feature, a probe direction), you want to understand its semantic meaning. There are four main approaches: (1) find real examples from a dataset that maximally activate the feature; (2) generate synthetic inputs via gradient ascent to maximise the feature; (3) steer the model by adding the feature direction to the residual stream and observing changed outputs; (4) use an LLM to automatically label max-activating examples.

Each approach has different failure modes. Max-activating examples can be dominated by frequency-of-occurrence rather than conceptual salience. Gradient ascent tends to produce high-frequency noise unless regularised. Steering can produce coherent text but the interpretation may be post-hoc. Automated LLM labelling can produce confident but incorrect labels if the labelling model lacks the right priors.

Gold standard

The gold standard is convergence across methods: max-activating examples, steering, and automated labelling all point to the same concept. Single-method evidence is suggestive but not conclusive.

Intuition First

Imagine you want to understand what a smoke detector is detecting. You could: (A) record all the times it went off and look for patterns; (B) slowly increase smoke concentration until it triggers; (C) play the detector's output through a speaker and listen. These correspond to max-activating examples, activation maximisation, and feature steering respectively.

Each method gives partial information. The detector might go off for dust (A), for precisely calibrated smoke patterns (B), or its output might sound like noise (C). You'd want all three to agree on "smoke" before concluding that's what it detects.

Technical Core

Max-activating dataset examples: Run a large corpus through the model. For each feature (neuron or SAE direction), record the activation value at every token position. Report the top-K token positions by activation. Display the surrounding context (±10 tokens). This is cheap and scales well but biases toward frequent contexts.

Activation maximisation (vision): Given a feature f and model φ, find input x* = argmax_x f(φ(x)) subject to regularisation constraints (total variation, L2 norm, frequency penalty). The gradient ∂f/∂x tells us which pixel changes increase the feature's activation. Iterate with gradient ascent.

Activation Maximisation Loop:
x ← random noise image
for t in range(T):
    h = model_up_to_layer(x)
    a = feature_activation(h)       # scalar
    grad = ∂a / ∂x
    x ← x + lr * grad               # gradient ascent
    x ← regularise(x)               # clip, blur, L2 penalty
return x

Feature steering (text): For a feature direction d ∈ ℝ^{d_model}, run the forward pass but at layer l add α·d to the residual stream at every token position. Observe how the generated text changes. Large α causes strong, sometimes incoherent changes. Small α may have no visible effect. The steering coefficient α is a free parameter.

# Feature steering in GPT-2
def steer_hook(module, input, output):
    output[0][:, :, :] += alpha * feature_direction
    return output

hook = model.transformer.h[9].register_forward_hook(steer_hook)
steered_output = model.generate(input_ids, max_new_tokens=30)
hook.remove()

Automated interpretability: Pass the top-20 max-activating examples to GPT-4 with the prompt: "The following text excerpts all strongly activate a particular feature of a language model. What concept do they share?" Collect the label. Validate by checking whether the bottom-20 activating examples lack that concept.

Worked Example

import torch
from transformers import GPT2Model, GPT2Tokenizer

model = GPT2Model.from_pretrained("gpt2")
tok   = GPT2Tokenizer.from_pretrained("gpt2")
model.eval()

# Step 1: collect activations for neuron 314, layer 9 MLP
corpus = [
    "The cat sat on the mat.",
    "Paris is the capital of France.",
    "Scientists discovered a new species.",
    "The stock market fell today.",
    "She played the piano beautifully.",
]
activations_store = []
hooks = []

def capture(module, inp, out):
    activations_store.append(out.detach().clone())

hook = model.transformer.h[9].mlp.register_forward_hook(capture)

for text in corpus:
    ids = tok(text, return_tensors="pt")["input_ids"]
    with torch.no_grad():
        model(ids)

hook.remove()

# Step 2: find top activating tokens for neuron 314
for i, (text, acts) in enumerate(zip(corpus, activations_store)):
    ids = tok(text)["input_ids"]
    neuron_acts = acts[0, :, 314]  # (T,) for neuron 314
    top_pos = neuron_acts.argmax().item()
    top_token = tok.decode([ids[top_pos]])
    print(f"Sentence {i}: max activation at '{top_token}' = {neuron_acts[top_pos]:.2f}")

Mechanistic Interpretation Layer

Text Models

Anthropic's monosemanticity work (Bricken et al. 2023) found SAE features with strikingly clear labels using automated interpretability. Feature #2663 activates for DNA sequences across all contexts (ATCG patterns, gene descriptions, molecular biology papers). Feature #31021 activates for text about bridges — musical bridges, architectural bridges, bridge as metaphor. The automated label matched human label in 73% of cases. The 27% mismatch rate corresponds mostly to features with correct labels but incomplete descriptions.

Vision Models

Olah et al. 2020 (Distill) showed that activation maximisation in InceptionNet produces curve detectors, frequency detectors, and multimodal neurons. Curve detectors in layer Mixed3a respond to edges at specific orientations. Feature visualisation using activation maximisation with Fourier regularisation produces clean, interpretable patterns for these neurons. The technique generalises to ViT but produces less clean visualisations due to attention's non-local receptive field.

Speech Models

Shain et al. 2020 and subsequent work visualises wav2vec2 features using reconstruction: given a feature direction, find audio segments that maximally project onto it. These segments correspond to phoneme-level features (high-frequency consonants for strident features, low-frequency energy for voiced features). Feature steering in text-to-speech models by adding a "fast speech" direction to the latent produces measurably faster speech. Caution: audio feature directions interact — adding a "high pitch" direction also affects duration and speech rate.

Hands-on Exercise

Label a GPT-2 MLP Neuron

  • 1Collect activations of neuron 314, layer 9 on 200 diverse sentences from a public dataset (e.g., first 200 sentences of Wikipedia introduction paragraphs).
  • 2Find the top-10 and bottom-10 token positions by activation value. Extract each with ±5 token context window.
  • 3Without looking at a label, write your hypothesis for what the neuron detects based on the top-10 examples.
  • 4Check whether the bottom-10 examples are consistent with your hypothesis (they should NOT exhibit the concept).
  • 5Now do feature steering: add 3× the neuron's output direction to the residual stream and generate text from a neutral prompt. Does the output shift toward your hypothesised concept?

Reflection: If the steering result contradicts the max-activating example hypothesis, which do you trust more, and why?

Stretch: Repeat for 5 different neurons and build a mini-labelling spreadsheet. What fraction were interpretable?

Real Model Artifact

Bricken et al. 2023 "Towards Monosemanticity" (Anthropic) trained a one-layer transformer on 8B tokens and applied SAE to the MLP layer. Of 4096 SAE features: 73% received a human label considered "clean"; 15% were labelled as "mixed" (partially interpretable); 12% were "unclear." The top-5 highest-activation-frequency features corresponded to: punctuation, capitalisation at sentence start, proper nouns, numbers, and function words — all high-frequency features. Rarer features were more concept-specific. The feature activation frequency followed a power law: a small number of features were active very frequently, and most features were active rarely.

Assessment Check

Scenario 1

Your top-10 max-activating examples for a feature all contain the word "bank" — but in different contexts: river bank, bank account, blood bank, memory bank. How do you determine which sense the feature represents?

Strong answer: The feature may be polysemantic (activating for all uses of "bank") or it may represent the abstract concept of "storage/repository" which unifies all senses. Tests: (1) check bottom-10 examples — do they lack all senses of "bank" or only one? (2) do feature steering — what text is generated when this feature is boosted? (3) compare activation strength across senses: if "river bank" consistently gives higher activation than "bank account", the feature may be sense-specific. (4) check if this feature co-activates with spatial or financial features (SAE feature co-occurrence analysis).

Scenario 2

Activation maximisation for a ViT neuron produces an image that looks like coloured noise rather than a recognisable object. Does this mean the neuron is uninterpretable?

Strong answer: No — activation maximisation without regularisation tends to produce adversarial-looking inputs that maximise a neuron but don't correspond to natural images. The neuron may still be interpretable via max-activating natural images. Try: (1) add total variation regularisation and L2 penalty; (2) use Fourier spectrum constraint; (3) switch to max-activating dataset examples. If those also show no pattern, the neuron may genuinely be polysemantic or encoding a high-frequency texture feature that isn't recognisable to humans.

Toy Models of Computation

Track 3: Interpretability
7 of 8
All previous Track 3 lessons
90 min
Text · Vision · Speech

Learning Outcomes

  • Explain the role of toy models as controlled interpretability environments
  • Describe grokking and what it tells us about generalisation and circuits
  • Train a 1-layer transformer on modular arithmetic and inspect its algorithm
  • Identify the Fourier-basis algorithm that transformers use for modular addition
  • Discuss what toy model findings do and do not generalise to large models
  • Design a toy model experiment to test a specific mechanistic hypothesis

Conceptual Setup

Toy models are small, fully tractable neural networks trained on simple, well-defined tasks. They are the fruit flies of mechanistic interpretability: their small size makes complete analysis tractable (you can visualise every weight, run every ablation), and the simplicity of the task means you can verify the discovered algorithm against the known correct algorithm.

The key paradigm in toy model research is grokking (Power et al. 2022): the phenomenon where a small transformer, trained on modular arithmetic, initially memorises the training data (100% train accuracy, ~chance test accuracy) then, after much more training, suddenly "groks" the underlying algorithm (test accuracy jumps from chance to 100%). This grokking event corresponds to a phase transition in the internal representations — from memorisation circuits to generalisation circuits.

The insights from toy models include: (1) transformers can implement elegant mathematical algorithms (Fourier analysis for modular arithmetic); (2) phase transitions in generalisation correspond to changes in circuit structure; (3) regularisation (weight decay) accelerates grokking by penalising memorisation circuits; (4) mechanistic analysis can fully reverse-engineer the algorithm learned by a toy model.

Intuition First

Imagine teaching a child to compute (a + b) mod 113. The child could memorise all 113² = 12769 pairs — like a lookup table. This works perfectly on the training set but requires enormous memory and doesn't generalise to new tasks. Alternatively, the child could learn the elegant algorithm: convert a and b to positions on a clock with 113 hours, add the angles, read off the result. This algorithm generalises to any values and uses much less memory.

Grokking shows that neural networks start with the memorisation strategy and, with sufficient training, discover the algorithmic strategy. The moment of "grokking" is when the network throws away the lookup table and implements the clock-arithmetic algorithm. Mechanistic interpretability allows us to verify this — and indeed, Nanda et al. 2023 showed that the post-grokking network literally computes Fourier components of a and b, multiplies them, and reads off the result.

Technical Core

Task: Given tokens a, b, =, predict (a + b) mod p. Model: 1-layer transformer, 2 attention heads, d=128. Training set: 30% of all (a,b) pairs; test set: remaining 70%.

The Fourier algorithm: After grokking, the model computes:

Embedding of a → Fourier components: e^{2πiωa/p} for key frequencies ω
Embedding of b → Fourier components: e^{2πiωb/p}
Attention: multiply Fourier components of a and b
  → e^{2πiω(a+b)/p}  [product-to-sum trigonometric identity]
MLP: extract the answer from the trigonometric pattern
  → argmax over 0..p-1 of correlation with target Fourier pattern

This is a genuinely elegant algorithm. The transformer has independently discovered something akin to the Number Theoretic Transform (NTT), a variant of the Fast Fourier Transform used in modular arithmetic.

Verifying the algorithm: After training, check that: (1) embedding vectors for a cluster by value of sin(2πa/p) — they form a circle in PCA; (2) the weight matrix W_logits has high cosine similarity with the Fourier basis vectors for the key frequencies; (3) ablating non-Fourier attention heads has no effect on performance; (4) the network uses exactly the "key frequencies" that are predicted by the algorithm.

Memorisation100%~5%Lookup table in embeddings
Transition100%50-90%Competing circuits

Worked Example

import torch
import torch.nn as nn
import numpy as np

p = 113  # prime modulus

# Minimal 1-layer transformer for modular addition
class ModularTransformer(nn.Module):
    def __init__(self, p, d=128, n_heads=2):
        super().__init__()
        self.embed = nn.Embedding(p + 1, d)  # +1 for = token
        self.pos   = nn.Embedding(3, d)       # positions: a, b, =
        self.attn  = nn.MultiheadAttention(d, n_heads, batch_first=True)
        self.mlp   = nn.Sequential(
            nn.Linear(d, d * 4), nn.ReLU(), nn.Linear(d * 4, d))
        self.unembed = nn.Linear(d, p)

    def forward(self, a, b):
        eq = torch.full((a.shape[0],), p, device=a.device)  # = token
        x  = torch.stack([a, b, eq], dim=1)                  # (B, 3)
        x  = self.embed(x) + self.pos(torch.arange(3, device=x.device))
        x, _ = self.attn(x, x, x)
        x  = x + self.mlp(x)
        return self.unembed(x[:, 2, :])  # predict from = position

# Generate all (a,b) pairs
all_a = torch.arange(p).repeat(p)
all_b = torch.arange(p).repeat_interleave(p)
all_c = (all_a + all_b) % p

# Train/test split (30/70)
perm = torch.randperm(p * p)
train_idx, test_idx = perm[:int(0.3 * p * p)], perm[int(0.3 * p * p):]

model = ModularTransformer(p)
opt   = torch.optim.AdamW(model.parameters(), lr=1e-3, weight_decay=1.0)
print("Training... (grokking typically occurs around epoch 500-1000)")

Mechanistic Interpretation Layer

Text Models

Toy model findings generalise to large LMs in specific ways. Induction heads — first identified in 2-layer toy models — were confirmed in GPT-2 and all subsequent transformers. The Fourier basis for modular arithmetic is present in GPT-2's numeric token representations. The phase transition in grokking mirrors the "task acquisition" phase transition observed when fine-tuning large models on reasoning tasks. The key limitation: toy model circuits are smaller and simpler — large model circuits may implement the same algorithm with more redundancy and distributed representation.

Vision Models

Toy vision experiments use small CNNs trained on synthetic datasets where the ground truth algorithm is known. For example, a CNN trained to detect circles implements approximate edge detection in layer 1 and curvature combination in layer 2 — verifiable because we know the geometry of circles. ViT toy models trained on synthetic shape datasets show that attention heads implement spatial grouping — attending to all patches of the same shape.

Speech Models

Toy ASR models trained on synthetic speech (rule-based TTS output) show cleaner circuit structure than models trained on natural speech. In a toy model trained to transcribe synthetic vowels, the first attention layer implements a spectral matching circuit: it attends from the output token position to the audio frame with the highest formant energy matching the target vowel's F1/F2 pattern. This is a clean example of a "query-from-output, key-from-audio" attention pattern.

Hands-on Exercise

Reproduce Grokking on Modular Addition

  • 1Implement the ModularTransformer from the worked example. Use p=113, d=128, 2 heads, weight_decay=1.0.
  • 2Train for 2000 epochs. Log train and test accuracy every 10 epochs.
  • 3Plot train and test accuracy over time. Identify the grokking epoch (when test acc jumps from below 20% to above 90%).
  • 4After grokking, apply PCA to the embedding matrix. Plot the embeddings coloured by value (0, 1, ..., 112). Do they form a circle?
  • 5Compute the Discrete Fourier Transform of the embedding matrix rows. Find the dominant frequency. Does it match a predicted key frequency (typically ω = 14 or ω = 35 for p=113)?

Expected output: Grokking at ~epoch 600 with weight_decay=1.0. Circle PCA visible. Dominant DFT frequency at ω=14 (or similar).

Reflection: What happens if you set weight_decay=0? Does grokking still occur? What does this tell you about the role of regularisation?

Stretch: Try modular multiplication instead of addition. Does the model still grok? What algorithm does it use?

Real Model Artifact

Nanda et al. 2023 "Progress measures for grokking via mechanistic interpretability" fully reverse-engineered a 1-layer transformer trained on modular addition with p=113. Key findings: (1) the model uses exactly the Fourier algorithm described above; (2) there are exactly 5 "key frequencies" (ω = 14, 35, 41, 42, 52); (3) the algorithm can be expressed as 40 equations; (4) ablating any non-key-frequency component has no effect on accuracy; (5) the embedding space is geometrically a union of concentric circles in 2D, one per key frequency. The paper provides full code and weight checkpoints enabling exact reproduction.

Assessment Check

Scenario 1

You train a toy transformer on modular addition and find no grokking — test accuracy stays near chance even after 5000 epochs. What are the most likely causes?

Strong answer: (1) Weight decay too low (0 or near 0): the memorisation circuit doesn't get penalised, so the generalisation circuit never forms. Fix: increase weight_decay to 0.5–2.0. (2) Training fraction too high (>60% of pairs): the memorisation circuit can fit enough pairs to avoid the need for generalisation. Fix: reduce to 30%. (3) Model too large: the model has enough capacity to memorise without needing the Fourier algorithm. Fix: reduce d to 64 or 32. (4) Learning rate too low: the optimisation is stuck. Fix: use learning rate warmup and then cosine decay.

Scenario 2

A colleague argues that toy model findings are irrelevant to understanding GPT-4 because GPT-4 is trained on completely different data and has billions of parameters. How do you respond?

Strong answer: Toy models are useful for discovering mechanism classes, not for claiming that GPT-4 uses exactly the same circuits. Specifically: (1) induction heads were found in toy models and confirmed in all tested transformers including GPT-2; (2) the Fourier algorithm for modular arithmetic was found in toy models and confirmed in GPT-2's number representations; (3) superposition was studied in toy models and confirmed in large models via SAEs. The appropriate claim is not "GPT-4 has this circuit" but "this class of mechanism is possible and worth looking for." The hypothesis-generation value of toy models is real even if the exact transfer is limited.

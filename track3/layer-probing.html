Learning Outcomes

This lesson develops a rigorous understanding of probing classifiers as a diagnostic tool for identifying linearly encoded features in transformer hidden states. You will leave able to design, run, and critically interpret probing experiments — including understanding when high probe accuracy can and cannot support causal claims.

  • Define a probing classifier and state the precise claim that high probe accuracy supports: linear decodability, not causal usage.
  • Implement a POS-tag probe using logistic regression on GPT-2 hidden states with scikit-learn.
  • Compute probe selectivity (probe accuracy minus control probe accuracy) to control for probe capacity artifacts.
  • Explain minimum description length (MDL) probing and why it is more rigorous than raw accuracy.
  • Describe the structural probe of Hewitt & Manning (2019) and what it tests about syntactic geometry.
  • Identify which layers contain which types of information in BERT-large according to Tenney et al. (2019).

Conceptual Setup

Layer probing (also called probing classifiers or structural probes) is the practice of training a simple classifier — typically a linear model — on top of a frozen model's hidden states, to test whether a specific linguistic or perceptual property is encoded in those representations.

The core claim probing supports: If a simple linear classifier can predict property X from hidden state h with accuracy significantly above chance, then property X is linearly decodable from h. This is a necessary (but not sufficient) condition for the model to use that property computationally.

The probe is trained on frozen representations — the transformer weights are never updated. Only the probe's own parameters (W, b) are trained. This ensures that any information discovered was present in the pre-trained representations, not learned by the probe training itself.

Critical limitation: Linear decodability does not imply causal usage. The model might encode a property without using it for any downstream computation — for example, encoding part-of-speech tags as a side effect of other computations, while using different features for its actual predictions. Establishing causal usage requires activation patching (Lesson 3).

Probing is also used for negative results: if a well-designed probe cannot exceed chance accuracy, the property is very likely not linearly encoded at that layer, which is itself informative about the model's representations.

Intuition First

Imagine you have a black box that takes sentences and outputs 768-dimensional vectors. You want to know if the box "knows" about part-of-speech tags. Here is how you find out:

  1. Collect 10,000 sentences with POS annotations (e.g., from Penn Treebank).
  2. Run each sentence through the black box and extract the hidden state vector for each word.
  3. Train a logistic regression: input is the 768-dimensional hidden state, output is the POS label.
  4. If the logistic regression achieves 95% accuracy on a held-out test set, the black box is encoding POS information — at least in a linearly accessible way.
What the probe tells you vs. what it doesn't: The probe tells you the information is there. It does not tell you what the model does with that information. Think of the probe as a metal detector — it can tell you there is metal in the wall, but not whether the metal is structural steel or a forgotten nail.

The choice of probe complexity matters. A simple logistic regression is more conservative — it can only find linear structure. An MLP probe can find nonlinear structure, but may also "work too hard," learning to solve the task using correlations that the transformer doesn't actually use. The field generally prefers linear probes for conservatism.

Technical Core

Probe Setup

For a hidden state h ∈ ℝd and label y ∈ {1,...,C}, a linear probe learns:

f(h) = Wh + b    where W ∈ R^(C × d), b ∈ R^C

Training minimises cross-entropy loss on frozen h vectors. Only W and b are updated. The transformer weights are frozen.

Probe Architecture Diagram

Input sentence: "The cat sat on the mat"
       |
Frozen Transformer (GPT-2 / BERT)
       |
Hidden States h[l] in R^(T x 768)
       |    <- frozen, not updated
Linear Probe: W in R^(C x 768)
       |    <- trained on labels
Predictions: POS, NER, parse depth, etc.

Selectivity: Controlling for Probe Capacity

A probe with many parameters may achieve high accuracy just because it memorises the training data. Selectivity controls for this:

Selectivity = probe_accuracy - control_probe_accuracy

The control probe has the same architecture but is trained on randomly shuffled labels. If the control probe gets 70% (due to dataset imbalance or model capacity), and the real probe gets 85%, selectivity = 15% — a more honest measure of how much the probe is learning from the representations.

Minimum Description Length (MDL) Probing

MDL probing (Voita & Titov 2020) measures the compression gain from using the representations to encode the labels, compared to a random baseline. It asks: how many bits does knowing h save you when communicating y?

MDL is more rigorous than accuracy because it penalises probes for the complexity of the information they needed to learn. A large gain = genuine encoding. A small gain = the probe is exploiting statistical artifacts.

Structural Probes (Hewitt & Manning 2019)

Structural probes test a specific geometric hypothesis: do parse-tree distances between words correspond to L2 distances in hidden state space? The probe learns a linear transformation B such that:

tree_distance(word_i, word_j) ≈ ||B·h_i - B·h_j||²

If such B can be found with low error, then syntactic structure is geometrically encoded in the transformer's hidden space.

Worked Example

A minimal POS probe on GPT-2 layer 6 using a toy noun/verb dataset:

import torch
import numpy as np
from transformers import GPT2Model, GPT2Tokenizer
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score

model = GPT2Model.from_pretrained("gpt2")
tok = GPT2Tokenizer.from_pretrained("gpt2")
model.eval()

# Toy POS probe: 10 nouns, 10 verbs
nouns = ["cat", "dog", "house", "tree", "book",
         "city", "river", "mountain", "cloud", "star"]
verbs = ["run", "jump", "build", "read", "write",
         "sing", "dance", "climb", "swim", "fly"]

def get_embedding(word, layer=6):
    inputs = tok(word, return_tensors="pt")
    with torch.no_grad():
        out = model(**inputs, output_hidden_states=True)
    return out.hidden_states[layer][0, 0, :].numpy()

X = [get_embedding(w) for w in nouns + verbs]
y = [0]*10 + [1]*10  # 0=noun, 1=verb

X_train, X_test = X[:16], X[16:]
y_train, y_test = y[:16], y[16:]

clf = LogisticRegression(max_iter=500)
clf.fit(X_train, y_train)
preds = clf.predict(X_test)
print(f"POS probe accuracy at layer 6: {accuracy_score(y_test, preds):.2%}")

# Compare across layers
print("\\nProbe accuracy by layer:")
for layer in range(13):
    X_all = [get_embedding(w, layer=layer) for w in nouns + verbs]
    clf_l = LogisticRegression(max_iter=500)
    clf_l.fit(X_all[:16], y[:16])
    acc = accuracy_score(y[16:], clf_l.predict(X_all[16:]))
    bar = "#" * int(acc * 20)
    print(f"  Layer {layer:2d}: {acc:.2%}  {bar}")

# Control probe: random label shuffle
import random
y_shuffled = y[:]
random.shuffle(y_shuffled)
clf_ctrl = LogisticRegression(max_iter=500)
clf_ctrl.fit(X[:16], y_shuffled[:16])
ctrl_acc = accuracy_score(y_shuffled[16:], clf_ctrl.predict(X[16:]))
print(f"\\nControl probe accuracy: {ctrl_acc:.2%}")
print(f"Selectivity at layer 6: {accuracy_score(y_test, preds) - ctrl_acc:.2%}")
Expected results: Layer 6 should achieve ~80–90% on this toy dataset. Control probe should be near 50% (random guessing for balanced classes). Selectivity should be 30–40 percentage points.

Mechanistic Interpretation Across Modalities

Text: Probing GPT-2 for Grammatical Number Agreement

Train a probe to predict subject noun number (singular/plural) from the hidden state at the subject noun position. Probe across all layers.

Expected finding: Accuracy peaks around layers 4–6, the zone where syntactic features are most accessible. Later layers show decreased accuracy as semantic and predictive information dominates.

Why it matters: If the model uses subject number to predict verb form (e.g., "the cats [are/is]"), the information must be present in the hidden state at the subject position. High probe accuracy is a necessary precondition for this mechanism to work.

Caution: High probe accuracy at the subject position does not confirm that the model routes this information to the verb position. That requires activation patching (Lesson 3). Probing only confirms the information exists.

Vision: Probing ViT for Object Presence

Train a linear probe on ViT hidden states ([CLS] token) to predict binary labels: "does this image contain a dog?" Probe each layer from 0 to 12.

  • Layer 2: ~65% accuracy — slightly above chance. Low-level features only.
  • Layer 8: ~82% accuracy — object parts becoming recognisable.
  • Layer 12: >90% accuracy — object identity strongly encoded.

What is measured: Binary classification accuracy per layer.

Caution: ViT was trained with a classification objective — probing for ImageNet classes is partially circular. Test with held-out properties not present in the training objective for more trustworthy results.

Speech: Probing wav2vec2 for Phoneme Identity

Pasad et al. (2021) extensively studied probing across all 24 layers of wav2vec2-large for multiple linguistic properties. Their key findings form a "probing profile":

  • Phoneme identity: Best predicted at layer 6/24 — relatively early, where acoustic features are still prominent.
  • Speaker identity: Peaks at layer 1 — almost immediately, suggesting the model separates speaker information early.
  • Word identity: Peaks at layer 15 — much deeper, where contextual and lexical information accumulates.

What is measured: Accuracy of logistic regression predicting phoneme label from per-frame hidden states.

Caution: Framewise phoneme labels require forced alignment to the audio signal. Alignment errors reduce probe accuracy independently of what the model encodes — particularly at phoneme boundaries.

Hands-on Exercise

Full Layer-wise Probing Experiment

Task: Run a systematic probing experiment across all 13 layers of GPT-2, measuring POS-tag decodability at each layer with proper controls.

  1. Collect 500 word-level examples with POS tags using NLTK (nltk.corpus.treebank) or spaCy. Balance classes.
  2. For each word, extract the in-context hidden state at all 13 layers using output_hidden_states=True. Use the word in a sentence, not in isolation.
  3. For each layer, train a logistic regression probe with 80/20 train/test split.
  4. Record accuracy per layer. Plot accuracy vs. layer depth as a line chart.
  5. Add a control probe line: same experiment but with randomly shuffled labels. This is your selectivity baseline.
  6. Identify the peak layer. Is it the same for all POS categories, or does it differ by category?

Expected output:

  • Accuracy rises from ~55% at layer 0 to peak ~85% at layers 4–5.
  • Slight decrease toward layer 12 as semantic/predictive content dominates over syntactic features.
  • Control probe stays near 20–30% (depending on class balance).

Reflection questions:

  • Why might probe accuracy decrease in the final layers? What information is competing with POS tags in layer 12?
  • If probe accuracy at layer 6 is higher than at layer 12, what does this tell you about how GPT-2 uses POS information?
Stretch challenge: Implement MDL probing using online coding (see Voita & Titov 2020). Train the probe incrementally on 1%, 2%, 5%, 10%, ... 100% of the data and compute the area under the accuracy curve. Compare MDL results to raw accuracy. Do they agree on which layer has the strongest encoding?

Real Model Artifact

Tenney et al. (2019): "BERT Rediscovers the Classical NLP Pipeline"
Probed BERT-large across all 24 layers for 8 NLP tasks using a "scalar mix" aggregation method.

Key findings from this landmark study:

  • POS tagging: Peaks at layers 4–6. Grammatical categories are encoded early and stably.
  • Constituency parsing: Peaks at layers 6–9. Phrase structure requires more processing than word-level tags.
  • Dependency parsing: Peaks at layers 8–11. Syntactic relationships between non-adjacent words encoded in middle layers.
  • Semantic role labeling: Peaks at layers 16–20. Requires understanding the meaning of predicates, not just syntax.
  • Coreference resolution: Peaks at layers 18–22. Requires discourse-level understanding across sentences.

This layer-ordering mirrors the classical NLP pipeline: POS → parsing → semantics → discourse. BERT appears to solve these tasks approximately in order of their traditional computational complexity. This is not guaranteed by the architecture — it is an emergent property of training on language data.

Assessment Check

Scenario 1: Probe Accuracy and Causal Claims

A probe achieves 92% accuracy predicting sentiment from GPT-2 layer 10. A researcher concludes "GPT-2 uses sentiment information to guide its predictions." Is this inference valid?

Show strong answer

No — the inference is not valid. Probe accuracy shows encoding, not causal usage. 92% accuracy means sentiment is linearly decodable from layer 10's hidden states. It does not mean the model's prediction mechanism uses that sentiment signal.

To establish causal usage, you would need activation patching: find a pair of inputs that are identical except for sentiment, patch the layer 10 hidden state from the positive-sentiment run into the negative-sentiment run, and check whether the model's output probabilities shift in the expected direction. Only if patching the sentiment-encoding dimensions changes the output can you claim causal usage.

A weaker version of the inference — "GPT-2 encodes sentiment, which is available for use" — is valid.

Scenario 2: Linear vs. MLP Probe Gap

You compare probe accuracy for "syntactic depth" using a linear probe (65%) vs. a 2-layer MLP probe (78%). How do you interpret the 13-point gap?

Show strong answer

The gap suggests syntactic depth is not purely linearly encoded. The MLP can access structure that the linear probe cannot — nonlinear combinations of hidden state dimensions that correlate with tree depth.

However, the MLP result is less trustworthy. A larger model may be picking up task-relevant correlations or fitting to dataset-specific artifacts rather than genuinely decoding the feature. Two important checks:

  • Run the MLP on randomly shuffled labels. If it achieves >70% there too, the 78% result is mostly a capacity artifact.
  • Compare MDL for both probes. The linear probe's MDL gain may be larger even with lower accuracy, indicating it is extracting more genuine information per parameter.

The linear probe's 65% is a conservative lower bound on the linearly encoded information. This is the number to report as the primary finding.
